# 產品需求文件 (PRD)：股票 LINE Bot

## 1. 產品概述 (Product Overview)
*   **專案背景 (Background)**：
    投資人需要快速且方便地獲取即時股市資訊。相比於開啟繁重且介面複雜的券商 App，LINE 是多數人最常使用的通訊軟體。透過 LINE Bot，使用者可以無縫地在日常對話中查詢股價、接收到價通知，大幅降低獲取資訊的門檻，並能第一時間掌握市場動態。
*   **產品目標與價值 (Objectives & Value)**：
    *   提供即時、準確的台股報價與基本資訊查詢。
    *   提供主動的「到價提醒」服務，讓使用者不錯過關鍵的買賣時機。
    *   建立個人化的「自選股清單」，方便每日盤中與盤後追蹤關注標的。
*   **目標受眾 (Target Audience)**：
    *   股市散戶、上班族投資人。
    *   希望透過簡單對話介面快速取得股市資訊，不習慣複雜看盤軟體的使用者。

## 2. 功能需求 (Functional Requirements)
*   **功能清單與優先級 (Feature List & Priorities)**：
    *   **P0 (核心功能)**：
        *   **即時股價查詢**：輸入股票代號（如 2330），回傳即時報價、單日漲跌幅、最高/最低價及成交量。
        *   **大盤資訊查詢**：快速查詢加權指數、櫃買指數及大盤成交總量。
    *   **P1 (進階功能)**：
        *   **自選股清單**：新增、刪除、查詢個人關注的股票清單，一次查看多檔標的狀態。
        *   **到價提醒 (Price Alerts)**：設定特定股票的目標價（大於或小於特定價格），當觸發時，系統主動透過 LINE Push Message 發送通知。
    *   **P2 (未來擴充)**：
        *   **個股新聞與基本面**：提供個股近期相關新聞連結或基本面指標（如 EPS、本益比、殖利率）。
*   **使用者故事 (User Stories)**：
    *   身為一個上班族，我想要在 LINE 輸入「2330」，以便於快速知道台積電現在的股價，而不需要打開券商 App。
    *   身為一個投資人，我想要設定「長榮股價大於 200 時通知我」，以便於在忙碌開會時也能把握出場時機。
    *   身為一個散戶，我想要把關注的股票加入「我的自選股」，以便於每天早上能一鍵快速看一眼所有關注目標的開盤狀況。
*   **業務規則與邊界條件 (Business Rules & Edge Cases)**：
    *   **無效輸入**：若輸入的股票代號不存在或 API 查無資料，需回覆友善的錯誤提示（例如：「查無此股票，請確認代號是否正確」）。
    *   **API 與推播限制**：免費版 LINE 官方帳號有每月推播訊息 (Push Message) 數量限制，到價提醒功能需考量訊息額度，或限制每人設定的提醒數量上限。
    *   **非交易時間**：在盤後查詢時，應明確標示該價格為「收盤價」而非盤中即時價。

## 3. 使用者體驗與流程 (UX/UI & Flow)
*   **使用者流程 (User Flow)**：
    1.  **查詢股價**：使用者輸入文字 `2330` -> Bot 呼叫股市 API -> Bot 回傳包含股價資訊的 Flex Message 卡片。
    2.  **設定提醒**：使用者點擊 Flex Message 上的「設定提醒」按鈕 (Postback) 或輸入指令 -> Bot 紀錄目標價並存入資料庫 -> 系統背景排程定期檢查股價 -> 若條件達成，發送 Push Message 通知使用者並將該提醒標示為已完成。
    3.  **自選股管理**：點選 Rich Menu「我的自選股」-> Bot 撈取資料庫清單並查詢最新報價 -> 以 Carousel Flex Message 呈現所有自選股現況。
*   **介面需求 (UI Requirements)**：
    *   **Rich Menu (圖文選單)**：常駐於聊天室下方，包含「我的自選股」、「大盤資訊」、「設定提醒」、「使用說明」等快速按鈕，方便點擊操作。
    *   **Flex Message**：以精緻的卡片格式呈現股價資訊。數字顏色需符合台灣股市習慣（漲為紅色、跌為綠色），並附帶「加入自選」或「設定提醒」的互動按鈕。

## 4. 非功能性需求 (Non-Functional Requirements)
*   **效能與架構考量 (Performance & Architecture)**：
    *   Webhook 回應時間必須在 2 秒內（LINE 官方強制規定）。若股市外部 API 查詢較慢，需考慮非同步處理或實作快取 (Cache) 機制。
    *   股價快取策略：熱門股票的報價可於伺服器端快取 10~30 秒，以減輕外部 API 的請求負擔與次數限制。
*   **安全性 (Security)**：
    *   Webhook 必須嚴格驗證 `X-Line-Signature`，確保請求確實來自 LINE 官方伺服器。
    *   使用者的 LINE User ID 及自選股清單等資料應存放於資料庫（如 SQLite 或 PostgreSQL）中並妥善保護。

## 5. 數據追蹤與驗收標準 (Metrics & Acceptance)
*   **衡量指標 (Success Metrics)**：
    *   DAU (日活躍用戶數) 及每日查詢指令的總量。
    *   每日設定「到價提醒」與「加入自選股」的互動次數。
    *   封鎖率 (Block Rate)：評估推播頻率是否過高導致使用者反感。
*   **驗收標準 (Acceptance Criteria)**：
    *   使用者輸入正確股票代號，可在 2 秒內收到格式化良好的 Flex Message 報價資訊，且數值與市場同步。
    *   使用者加入/刪除自選股後，查詢自選股清單能正確反映最新狀態。
    *   系統背景排程能準確在股價達標時，發送 LINE 提醒訊息給該使用者。
*   **排除範圍 (Out of Scope)**：
    *   本次開發**不包含**實際下單交易功能（因涉及券商 API 授權與金管會法規問題）。
    *   本次開發**不包含**複雜的技術線圖 (如 K線圖) 動態繪製與互動，僅提供純數字或簡單的走勢圖片。
