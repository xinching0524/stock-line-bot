# 系統手動測試清單 (Test Cases)

## 1. 測試環境確認
*   **後端環境**：確認 `uvicorn backend.main:app --reload` 能夠正常啟動在 `http://localhost:8000` 沒有任何 Error 報錯。
*   **資料庫**：確認存放目錄中有 `data/temple.db`，並且已經執行過 `python backend/seed.py` 以載入初始的籤詩庫。
*   **前端環境**：開啟瀏覽器載入首頁 `http://localhost:8000`，應該正常顯示具有「帳號」與「密碼格」的 Glassmorphism 登入介面。

## 2. 功能性測試案例 (Functional Test Cases)

### TC-01: 使用者帳號註冊 (User Registration)
*   **前置條件**：無。
*   **測試步驟**：
    1. 在登入畫面的稱呼欄位輸入「測試信眾」。
    2. 密碼欄位輸入一組大於 4 碼的密碼（如「123456」）。
    3. 點擊「註冊」按鈕。
*   **預期結果**：系統下方顯示綠色的「註冊成功，請登入！」提示訊息。
*   **實際結果**：[請填寫]

### TC-02: 使用者登入系統 (User Login)
*   **前置條件**：已完成 TC-01 的註冊。
*   **測試步驟**：
    1. 在同一個登入畫面，保留剛剛註冊成功的帳號與密碼。
    2. 點擊「登入」按鈕。
*   **預期結果**：成功登入並切換顯示，上方顯示「信士：測試信眾」，並且畫面中央預設在「求神問卜」的介面。
*   **實際結果**：[請填寫]

### TC-03: 進行求籤體驗 (Draw Poem)
*   **前置條件**：成功登入。
*   **測試步驟**：
    1. 在「求神問卜」畫面的輸入框上填寫求問之事（例如：求接下來一個月的事業運）。
    2. 點擊「搖動籤筒」按鈕。
*   **預期結果**：按鈕短暫變為「搖動中...」，隨後下方會彈出籤詩卡片，包含籤頭、吉凶狀態（例如：大吉）、古文原文與白話文解析。
*   **實際結果**：[請填寫]

### TC-04: 香油錢捐獻 (Donation)
*   **前置條件**：成功登入。
*   **測試步驟**：
    1. 點選畫面上方的「香油捐獻」進入頁籤。
    2. 點選金額按鈕切換為 `NT$ 500`，下方輸入框寫入：「闔家平安，業績長紅」。
    3. 點選「確認供奉」。
*   **預期結果**：送出後按鈕下方會顯示綠顏色的「感謝您的發心，已記錄捐獻！」提示文字，並且該文字會在 3 秒後自動隱藏。

### TC-05: 查看參拜與歷史紀錄 (History Tracking)
*   **前置條件**：需至少執行過一次 TC-03 求籤與 TC-04 捐獻。

## 3. 異常與邊界測試 (Edge Cases)

### EC-01: 短密碼與格式防護測試
*   **測試步驟**：回到登入首頁，嘗試只輸入帳號以及少於 4 個字的密碼（如 123），接著點擊註冊。
*   **預期結果**：系統前端即攔截，應顯示紅字「請輸入名稱與至少 4 碼密碼」。

### EC-02: 重複註冊阻擋測試
*   **測試步驟**：輸入剛剛已經註冊過的帳號名稱「測試信眾」與一組密碼，點擊「註冊」。
*   **預期結果**：因違反資料庫 Unique Constraints，在發送 Request 後系統會顯示「此稱呼已被註冊，請換一個或登入」的 HTTP 400 例外提示。

### EC-03: 錯誤密碼登入測試
*   **測試步驟**：輸入正確已註冊的帳號名稱「測試信眾」，但刻意填寫錯誤的密碼，點擊「登入」。
*   **預期結果**：登入失敗，按鈕下方出現紅色的「稱呼或密碼錯誤」警告。

### EC-04: 未附帶使用者狀態調用
*   **測試步驟**：若是透過 API 工具 (如 Postman)，在 Payload 中傳入不存在資料庫中的 username 去呼叫 `/api/draw` 介面。
*   **預期結果**：後端的依賴函式 `get_user` 必須能成功攔截並回覆 HTTP Status Code 401 `無效的使用者`。
